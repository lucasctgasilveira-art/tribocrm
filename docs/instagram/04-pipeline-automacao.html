Plano de Comunicação · Bloco 4 de 5 · Pipeline de Automação

Pauta YAML entra. PNG no tamanho nativo do Instagram sai. Em 1.4 segundos.

Pipeline Node + Puppeteer em marketing/instagram/: lê YAML do calendário, carrega o template HTML certo, substitui os pontos editáveis e captura screenshot do .canvas em 1080×1350 ou 1080×1920. Veio com 5 pautas-exemplo do calendário Mês 1 prontas pra rodar.

01 Arquitetura

Fluxo end-to-end de uma pauta até virar PNG.

posts/2026-05-04_funil_01-capa.yaml
↓ js-yaml.load
template: carrossel-metodo-capa
↓ render-post.mjs
templates/carrossel-metodo-capa.html
↓ Chromium headless · page.goto(file://...)
substitui [data-edit="X"] pelo valor de data.X
↓ document.fonts.ready
canvas.screenshot()
↓ no tamanho nativo · 1080×1350 ou 1080×1920
output/2026-05-04_funil_01-capa.png

02 Setup (uma vez)

Instala Puppeteer (~150MB, baixa Chromium) + js-yaml.

$cd marketing/instagram
$npm install

Engines: Node ≥ 18 (usa import, parseArgs nativo, top-level await).

Dependências:

  • puppeteer ^23.0.0 — Chromium headless + API de screenshot
  • js-yaml ^4.1.0 — parse dos YAMLs de pauta

04 Anatomia de uma pauta YAML

Estrutura mínima: template + data. slug é opcional (default: nome do arquivo YAML).

# Pauta: Slide 1 do carrossel "Os 4 momentos do funil"
# Calendário: SEG 04/05/2026 · Pilar Método
template: carrossel-metodo-capa
slug: 2026-05-04_funil_01-capa
data:
  pilar: "Método"
  titulo: |
    Os 4 momentos do funil<br>
    que sua PME tá <span class="t-orange">pulando</span>.
  slide-number: "01 / 08"
  handle: "@tribocrmoficial"

Conteúdo HTML inline é permitido

O pipeline usa innerHTML, então você pode incluir:

  • <br> pra quebra de linha
  • <span class="t-orange">palavra</span> pra destacar palavra em laranja
  • <span class="cta-emoji">🔥</span> pra emoji no CTA
  • <strong> pra ênfase pontual

YAML multilinha com | preserva quebras visuais e permite HTML limpo no editor.

06 Como o pipeline funciona internamente

scripts/render-post.mjs faz, por pauta:

  1. Lê o YAML com js-yaml.load. Valida que tem template e data.
  2. Resolve o template em templates/<template>.html. Se não existir, falha.
  3. Lança Chromium headless via Puppeteer (viewport 1200×2000 — cobre feed e story).
  4. Carrega o template via file:// URL com waitUntil: 'networkidle0'.
  5. Injeta CSS de produção que desabilita o .preview-stage scaling — canvas renderiza em tamanho nativo, não escalado.
  6. Substitui cada [data-edit="X"] pelo valor de data.X via page.evaluate.
  7. Espera fontes com document.fonts.ready — DM Sans precisa estar carregada antes do screenshot.
  8. Mede o .canvas com boundingBox() pra reportar dimensão real.
  9. Captura screenshot do elemento .canvas (não da viewport inteira) → PNG limpo.
  10. Salva em output/<slug>.png.

render-batch.mjs faz o mesmo em loop, reusando a instância do browser.

07 Troubleshooting

"Failed to launch the browser process"
Puppeteer baixa o próprio Chromium na instalação. Se falhou, rode:
npx puppeteer browsers install chrome
Fonte DM Sans aparece como serif/Times
O tokens.css puxa Google Fonts via @import. O pipeline espera document.fonts.ready, mas em rede lenta pode não bastar — aumente o setTimeout em render-post.mjs:90 (atualmente 250ms) pra 500-1000ms.
Símbolo D não aparece no PNG
Confirme que _assets/symbol-d.svg existe e o template referencia ../_assets/symbol-d.svg.
Texto cortado / sobrando da peça
Conteúdo do YAML é maior que o template comporta. Reduz o texto ou ajusta o tamanho da tipografia no template via override de classe.
Como adicionar template novo
1. Cria templates/<novo>.html seguindo a convenção (link tokens.css + classe .canvas + elementos com data-edit).
2. Cria YAML em posts/ com template: <novo>.
3. Renderiza. Não precisa tocar no render-post.mjs — ele é genérico.
